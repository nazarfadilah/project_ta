========================================================================
LIST TABEL DAN FIELD UNTUK MODEL DATA (CDM, LDM, PDM)
Sistem Informasi Peminjaman Ruangan dan Sarana (SIPRASA)
========================================================================

Dokumen ini memuat daftar tabel dan field yang membedakan konsep CDM (Conceptual Data Model), LDM (Logical Data Model), dan PDM (Physical Data Model) serta rincian lengkap kamus data dari keseluruhan database.

------------------------------------------------------------------------
1. PERBEDAAN CDM, LDM, DAN PDM
------------------------------------------------------------------------

* CONCEPTUAL DATA MODEL (CDM)
  - Bersifat konseptual, berfokus pada representasi entitas bisnis dan hubungannya.
  - Tidak memuat Foreign Key (FK) sebagai kolom. Relasi digambarkan sebagai garis hubungan asosiatif langsung antar entitas.
  - Tipe data bersifat umum/logikal (misal: Text, Number, Date).
  - Kolom metadata teknis (seperti created_at, updated_at) tidak disertakan.

* LOGICAL DATA MODEL (LDM)
  - Menjembatani CDM dengan PDM.
  - Mulai memetakan kolom Foreign Key (FK) hasil dari resolusi relasi antar entitas.
  - Tipe data dispesifikasikan lebih rinci namun masih independen dari DBMS tertentu (misal: String, Integer, DateTime, Boolean).
  - Mengidentifikasi opsionalitas (Nullability) dan Primary/Foreign Key.

* PHYSICAL DATA MODEL (PDM)
  - Representasi fisik database riil yang siap diimplementasikan pada DBMS target (dalam hal ini MySQL/MariaDB).
  - Menggunakan tipe data spesifik DBMS (misal: VARCHAR, BIGINT UNSIGNED, ENUM, DECIMAL, TIMESTAMP).
  - Memuat relasi Foreign Key lengkap dengan aksi referensial (seperti ON DELETE CASCADE atau ON DELETE SET NULL).
  - Menyertakan seluruh kolom metadata teknis/log seperti created_at, updated_at, atau createdAt, updatedAt.
